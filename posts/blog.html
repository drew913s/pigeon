🕊️ Pigeon

Agent orchestration without orchestrators.

Every agent framework solves coordination with more code. More abstractions. More runtime state. More invisible failure modes. Kubernetes. Airflow. Celery. LangGraph. CrewAI. Layers on layers.

We solved it with mv.

The Idea

A pigeon is a .md file that moves between directories. Each directory is an agent. When the file lands, the agent activates. When it's done, it moves the file. That's the entire orchestration.

loft/           → home base (born here, retire here)
researcher/     → station.py
writer/         → station.py
editor/         → station.py
publisher/      → station.py

No orchestrator process. No message queue. No database. The file's location is the state. The file's content is the context. The file's movement is the coordination.

How It Works

Each station checks for a pigeon in its directory. If one is there, it reads it, does its work, writes what it did, and moves the file to the next station:

from pigeon import check, land, stamp, send

def run():
    pg = check("editor")          # File in my directory?
    if not pg: return             # No file, no work.
    content = land(pg)            # Read it.
    # ... do your work ...
    stamp(pg, "editor", "Fixed 3 tone issues")
    send(pg, "reviewer")          # Move it.

Every station is this pattern. No exceptions.

The Pigeon File

Every pigeon is human-readable markdown. cat to debug. ls loft/ for history.

# PG-001 — Quarterly Report
**Created:** 2026-03-22

## Plan
Research Q1 metrics, write summary, publish to blog.

## State
- articles: 12
- approved: 11

## Journey
| When | Agent | What Happened |
|------|-------|---------------|
| 14:00 | loft | Born |
| 14:01 | researcher | Found 12 topics |
| 14:30 | writer | Wrote 12 articles |
| 14:45 | editor | Fixed 3 tone issues |
| 15:00 | publisher | Published 11 to prod |

Properties

Multiple pigeons = parallel jobs. PG-001 at editor, PG-002 at writer. Independent.

Remove the file = stop the job. Put it back = resume from where it stopped.

Add an agent. New directory, new station.py. Existing agents don't change.

Any language. If it can read and move files, it can be a station.

Git friendly. Pigeons are text files. Version history for free.

Where It Came From

We built a 5-station AI training factory — research council, architecture designer, data encoder, GPU trainer, model evaluator. 17 brain-folder agents across 6 machines. We tried orchestrator scripts, signal files, polling loops, JSON state machines. Each one added complexity. Each one broke.

Then we asked: what's the simplest possible coordination?

Put a file where they can see it.

The pigeon pattern emerged from that question. We deleted 353 lines of orchestrator code and replaced them with shutil.move(). The system has been running since.

When NOT to Use This

Sub-millisecond latency. Millions of concurrent jobs. Stateless request/response. For those, use a real queue or HTTP.

For everything else: code breaks. Files don't.
